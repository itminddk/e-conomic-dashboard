How to Use Claude Code

Claude Code is an agentic CLI tool that reads, edits, and runs code in your terminal — with full access to your codebase.

1 Installation

Install Claude Code globally via npm. You need Node.js 18+ and an Anthropic API key.

terminal 
$ npm install -g @anthropic-ai/claude-code

Then authenticate with your Anthropic API key:

terminal
$ export ANTHROPIC_API_KEY=sk-ant-...
Tip: Add the export to your ~/.zshrc or ~/.bashrc so it persists across sessions.

2 Starting Claude Code

Navigate to your project directory and run claude:

terminal
$ cd my-project
$ claude

Claude will read your project structure and be ready to help. You can also pass a one-shot prompt directly:

terminal
$ claude "explain the architecture of this project"

3 Basic usage

Claude Code can perform all of these actions autonomously inside your project:

Read files

Reads any file in your project to understand context before making changes.

Edit files

Makes precise edits — from fixing a typo to refactoring entire modules.

Run commands

Executes shell commands like pytest, npm test, or git diff to verify changes.

Search codebase

Finds functions, classes, and patterns across your entire project with ripgrep.

Create files

Scaffolds new components, tests, configs, and more from scratch.

4 Prompting effectively

Be specific and task-oriented. Claude works best when you describe the outcome you want.

Fix the bug where login fails when email contains a plus sign
Add pagination to the /users endpoint, 20 items per page
Write tests for all public methods in src/auth/token.py
Refactor the database connection to use a connection pool
Explain how the request lifecycle works in this app
Find everywhere we're not handling errors and fix them
Tip: You don't need to point Claude to specific files — it will search and find the relevant code itself.

5 Slash commands

Type / inside the Claude Code prompt to access built-in commands:

/help

Show all available commands and keyboard shortcuts.

/clear

Clear the conversation history and start fresh.

/compact

Compress the conversation to free up context window space.

/review

Ask Claude to review the current diff or open PR.

/commit

Generate a commit message and commit staged changes.

/init

Create a CLAUDE.md file with project context and conventions.

/cost

Show the token usage and cost for the current session.

/bug

Open a GitHub issue to report a Claude Code bug.

6 Permissions

Claude Code asks for approval before running commands or making changes. You can control the permission level:

Default mode

Claude asks before running shell commands. File reads and edits are auto-approved.

--dangerously-skip-permissions

Auto-approves everything. Use only in trusted, sandboxed environments.

Note: Destructive actions like rm, force-push, or dropping databases will always prompt for confirmation regardless of mode.

7 Git integration

Claude Code understands your git history and can help with the full development workflow:

Review the changes I've made since the last commit
Write a commit message for my staged changes
Create a pull request for this branch
What changed between v1.2 and v1.3?

Hooks

Hooks let you run shell commands automatically in response to Claude's actions — before or after file edits, tool calls, and more. Configure them in ~/.claude/settings.json:

~/.claude/settings.json
{
  "hooks": {
    "PostToolUse": [
      {
        "matcher": "Write|Edit",
        "hooks": [{ "type": "command", "command": "ruff format $FILE" }]
      }
    ]
  }
}

MCP servers

Model Context Protocol (MCP) servers extend Claude Code with new tools — databases, APIs, Slack, Google Calendar, and more. Add them in ~/.claude/settings.json:

~/.claude/settings.json
{
  "mcpServers": {
    "my-db": {
      "command": "npx",
      "args": ["@modelcontextprotocol/server-postgres"],
      "env": { "DATABASE_URL": "postgres://..." }
    }
  }
}

Pro tips

CLAUDE.md

Add a CLAUDE.md at your project root with conventions, commands, and context. Claude reads it automatically every session.

Be outcome-focused

Describe what you want to achieve, not how to do it. Let Claude figure out the implementation.

Iterate in session

Claude remembers context within a session. Follow up with corrections or extra requirements without re-explaining everything.

Use /compact often

For long sessions, run /compact to summarize history and keep the context window healthy.

Review before approving

Always read the diff before approving shell commands or file writes, especially in production environments.
